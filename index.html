React Hooks

  • Manage state in function components
  • Introduced in React v16.8
  • Opt-in, no breaking changes

useState

  • Preserves state between renders
  • Returns the current state each time the function is called.
  • Can be used multiple times in a single component.

https://reactjs.org/docs/hooks-state.html

const [state, setState] = useState(initialState);

useEffect

  • Can be used to replace lifecycle hooks
  • Runs a method when data changes
  • Pass an array to only run when certain data changes

https://reactjs.org/docs/hooks-effect.html

useEffect(() => {
  console.log('Component Mounted');
}, []);

useEffect(() => {
  loadData(key);
}, [key]);

And More!

Rules for Hooks

  • Only call hooks at the top level of the function
    • Do not use hooks in if statements or loops
    • Hooks must be called in the same order
  • Hooks can only be used in functional components

https://reactjs.org/docs/hooks-rules.html

export function Hello(props) {
  const [date, setDate] = useState(null);

  useEffect(() => {
    setDate(new Date());
  }, []);

  return (
    <div>Hello {props.name}. Today is {date.toString()}</div>
  );
}

"Lifting state up"

function HelloWorld(props) {
  return (
    <div>
      Hello {props.name}. 
      Today is {props.date.toString()}
    </div>
  );
}

function CurrentDate(props) {
  const [date, setState] = useState(null);
  useEffect(() => setState(new Date()), []);

  return (
    <HelloWorld name={props.name} date={date} />
  );
}

https://reactjs.org/docs/lifting-state-up.html

Render Props

https://reactjs.org/docs/render-props.html

function CurrentDate(props) {
  const [date, setState] = useState(null);
  useEffect(() => setState(new Date()), []);
  return props.children(date);
}

function HelloWorld(props) {
  return (
    <div>Hello {props.name}. Today is {props.date.toString()}</div>
  );
}

function App(props) {
  return (
    <CurrentDate>
      {date => <HelloWorld name="Dave" date={date} />}
    </CurrentDate>
  );
}

Context

const AuthContext = React.createContext({ username: null });
const AuthContextProvider = AuthContext.Provider;

function Profile(props) {
  const authContext = useContext(AuthContext);
  return (
    <div>{authContext.username}</div>;
  );
}

function App() {
  const [auth, setAuth] = useState({ username: 'JohnDoe' });
  return (
    <AuthContextProvider value={auth}>
      <Profile />
    </AuthContextProvider>
  );
}
const AuthStateContext = React.createContext();
const AuthDispatchContext = React.createContext();

function AuthProvider(props) {
  const [auth, setAuth] = useState(null);
  const dispatch = (un) => setAuth({ username: un });
  return (
    <AuthStateContext.Provider value={auth}>
      <AuthDispatchContext.Provider value={dispatch}>
        {props.children}
      </AuthDispatchContext.Provider>
    </AuthStateContext.Provider>
  );
}

When to use Context

  • The same data is needed by many components
  • Good for data that does not change often
  • Will make component re-use more difficult
  • Can be difficult to track state and changes

https://reactjs.org/docs/context.html#when-to-use-context

Redux

Redux

  • A predictable state container for JavaScript apps.
  • Provides a single source of truth for state.
  • Separates state from your view.

https://redux.js.org/introduction/getting-started

Actions

  • Type - Static string used to identifiy the action. Every thing else is optional
  • Payload - The data of the action. Generally used by the reducer
  • Error - Indicates if the action is an error
  • Meta - Additional information on the action.
const action = { type: 'Update Timestamp' };

Reducers

  • Accepts the previous state and action to return the new action
  • Multiple reducers can be combined into a single reducer with combineReducers
function rootReducer(state = {}, action) {
  switch(action.type) {
    case 'Update Timestamp': 
      return { ...state, timestamp: action.payload };
    default: 
      return state;
  }
}

Selectors

  • Function that accepts state and can return computed values.
  • Not always used, but a good way to get information from state.
function selectTimestamp(state) {
  return state.timestamp
    ? state.timestamp.toString()
    : 'Unknown';
}

Middleware

const loggerMiddleware =
  ({ dispatch, getState }) =>
  (next) =>
  (action) => {
    const result = next(action);
    console.log(action);
    return result;
  };
  • Can intercept actions to modify how they are handled or dispatch additional actions.
  • Provides the ability to customize redux in many ways
//reducer
function rootReducer(state, action) {
  switch(action.type) {
    case 'Update Timestamp': return { ...state, timestamp: action.payload }
    default: return state;
  }
}

//selector
function selectTimestamp(state) {
  return state.timestamp ? state.timestamp.toString() : 'Unknown';
}

function TimestampDisplay(props) {
  const timestamp = useSelector(selectTimestamp);
  const dispatch = useDispatch();

  const update = () => {
    dispatch({ type: 'Update Timestamp', payload: new Date() });
  };

  return (
    <>
      <div>{timestamp}</div>
      <button onClick={update}>Update</button>
    </>
  );
}

const store = createStore(rootReducer);
ReactDOM.render(
  <Provider store={store}>
    <TimestampDisplay />
  </Provider>,
  document.getElementById('root')
);

When to use Redux?

  • When you have complex state
  • Take advantage of the tooling
  • Does introduce more boilerplate
  • Adds indirection to state management

What is complex state?

  • Can be changed by multiple components
  • Is used by multiple components
  • There are relationships in your state

What is the "right" way to manage state?

Custom Hooks

const [count, increment, decrement] = useCounter()
  • A custom hook is any function that uses hooks.
  • They follow the same rules as hooks
  • Generally prefixed with "use"
function useCounter() {
  const [count, setCount] = useState();
  return [
    count, 
    () => setCount(count + 1), 
    () => setCount(count - 1)
  ];
}
//reducer
function count(state = 0, action) {
  switch(action.type) {
    case 'increment': return state + 1;
    case 'decrement': return state - 1;
    default: return state;
  }
}

//selector
const selectCount = (state) => state.count;

function useCount() {
  const count = useSelector(selectCount);
  const dispatch = useDispatch();

  return [
    count, 
    () => dispatch({ type: 'increment' }), 
    () => dispatch({ type: 'decrement' })
  ];
}
  • Can be expressive with your state
  • Separates your state from your components
  • Use the approach that makes sense for that state

Why custom hooks?

  • If state is not shared, use component state
  • Render Props is a good pattern to share state in small sections of your app.
  • Context can be used for shared state that is used by many components, but not updated often
  • Redux can be used for shared complex state
  • Create hooks to manage shared state to simplify your components
